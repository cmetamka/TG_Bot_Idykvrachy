# Иду к врачу — телеграм-бот и Mini App

Бот для клиники «Иду к врачу»: приветствие, переход в мини-приложение, разделы «Подготовка» (стоматолог, сдача крови), запись к врачу, «Написать нам».

## Установка

```bash
cd c:\Prog\tgbot_vrach
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Токен бота уже прописан в `.env`. При необходимости скопируйте `.env.example` в `.env` и укажите `BOT_TOKEN` и `MINIAPP_URL`.

## Запуск

### Вариант 1: Всё одной командой (рекомендуется)

Запускает и Flask, и бота, и при необходимости поднимает HTTPS-туннель (ngrok), чтобы кнопка «Начать» в Telegram открывала Mini App:

```bash
py -3 -m pip install -r requirements.txt
py -3 run.py
```

**Запуск без ngrok** (только бот и приложение на localhost, кнопка «Начать» в Telegram не откроет Mini App):

```bash
py -3 run.py --no-tunnel
```

Или в `.env` добавьте `TUNNEL=0` и запускайте как обычно. Mini App тогда открывайте в браузере: http://localhost:5000

### Вариант 3: Туннель через Serveo (ssh)

Если ngrok недоступен, можно использовать SSH‑туннель через `serveo.net`.

Требования:
- установлен ssh‑клиент (OpenSSH Client на Windows — можно поставить через «Дополнительные компоненты Windows»);
- доступ к `serveo.net` не заблокирован.

Запуск:

```bash
py -3 run_serveo.py
```

Скрипт:
- поднимает Flask на порту 5000,
- выполняет `ssh -R 80:localhost:5000 serveo.net`,
- читает выданный HTTPS‑адрес вида `https://xxx.serveo.net` и ставит его в `MINIAPP_URL`,
- запускает бота, и кнопка «Начать» открывает Mini App по этому адресу.

Чтобы туннель работал, зарегистрируйтесь на [ngrok.com](https://ngrok.com) (бесплатно), возьмите токен и добавьте в `.env` строку:

```
NGROK_AUTH_TOKEN=ваш_токен
```

После этого при запуске `run.py` в консоли появится HTTPS-адрес Mini App, и кнопка «Начать» в боте будет открывать приложение.

Если вы установили ngrok, но `run.py` всё равно пишет про скачивание/таймаут, значит команда `ngrok` не находится в PATH. Тогда укажите в `.env` полный путь к `ngrok.exe`:

```
NGROK_PATH=C:\путь\к\ngrok.exe
```

### Вариант 2: По отдельности

1. **Сервер мини-приложения:** `py -3 app.py` — приложение на http://localhost:5000  
2. **Бот:** в другом терминале `py -3 bot.py`  
Для работы кнопки «Начать» нужен HTTPS (например ngrok или свой сервер с SSL).

### 3. Пересылка «Написать нам» админу

Узнайте свой Telegram ID (например через [@userinfobot](https://t.me/userinfobot)) и укажите в `.env`:

```
ADMIN_CHAT_ID=ваш_id
```

Сообщения от пользователей после нажатия «Написать нам» будут пересылаться в этот чат.

## Что сделано

- **Бот:** приветствие по `/start` и при любом сообщении, кнопка «Начать» (Web App), пересылка сообщений админу.
- **Mini App:** экран «Добро пожаловать», главное меню (Подготовка, Запись к врачу, Сервис «Иду к врачу», Написать нам), раздел «Подготовка» с направлениями «Поход к стоматологу» и «Сдача анализа крови».
- **Поход к стоматологу:** мультфильм (ссылка на Rutube), социстория (два слайда с текстом), игра-тренажер (ссылка подставится позже), рекомендации для родителей (заглушка).
- **Сдача крови:** экран «Материалы для адаптации» в том же стиле (мультфильм, социстория, игра — пока заглушки «Скоро»).

Ссылку на игру-тренажер можно позже прописать в `static/index.html` в элементе с `id="dentist-game-link"`. Ссылку на внешний сервис «Иду к врачу» — в обработчике кнопки «Сервис» в `static/js/app.js`.
